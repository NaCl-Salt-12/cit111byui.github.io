(W06 - Creating Insert Statements File):

    • As a class, use the movie data to create a file that contains all the insert statements they will need to populate the database. They will do something similar for their homework and this is good practice. You can populate a few tables as a class and then they can try some on their own or in groups. It is always confusing students on how to insert the linking tables data, so go over these examples well.

    • Show them how to Forward Engineer their ERD from last week with Database - Forward Engineer (have them check the "DROP objects before each CREATE object" and "Generate DROP SCHEMA" -- this will help if they end up Forward Engineering multiple times). They will need to do this to create their empty art database for their homework. Then they will use their INSERT statements file to populate this database.
    • Show them how to create a dump file with Server - Data Export - Self-contained file. They will need to know how to do this after they have all the data in their database. It is one of the files they will need for their homework.

  • End of the Week
    • The homework due Saturday will be
      • creating a SQL file with all the insert commands to populate the art database. Once their database has data they will export a dump file.

Grading/Feedback

  • Instructor: The check and quiz are auto-graded. The homework will be instructor or TA graded. Look at their SQL insert commands file and their dump file. In the insert file they could have used different types of insert commands to get the data into the database. In the dump file you will check that they have the data actually went into the tables properly. (I found from past experience that even though they can enter data into the result set area in Workbench, it's usually best to have them create SQL insert statements. This is because if they mess up their database it's much easier to just drop that database and reverse engineer again and start over and if they have these inserts saved, they can edit and re-run them much easier than having to re-enter all the data into the database again.)
